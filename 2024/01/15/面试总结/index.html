框架

SpringMVC

拦截器和过滤器的区别

  • 拦截器是基于反射机制实现，过滤器通过函数回调实现。
  • 拦截器不依赖servelet容器，过滤器依赖servelet容器。
  • 拦截器只对Action起作用，过滤器对所有请求起作用。
  • 在Action的生命周期中，拦截器可以多次调用，过滤器只在容器初始化时调用一次。

Mysql的分页

Mysql索引

什么是索引？

本质可以看成是一种排序好的数据结构，用来提高数据库查询效率。

索引的类型
  • 普通索引
  • 主键索引
  • 唯一索引
  • 全文索引
索引的优缺点？

优点：

  • 提高数据库查询效率
  • 可以创建唯一性索引，来确保表中字段数据的唯一性。

缺点：

  • 在进行增删改操作的时候，如果字段有索引，也要动态的修改索引，降低了SQL语句的执行效率。
  • 索引要占用一定的物理空间。
B树和B+树的区别？
  • B树所有节点都存放key和value，B+树叶子节点存放key和value，非叶子结点只存放key。
  • B+树的叶子节点有指针指向相邻的叶子节点。

Mysql优化

  • 对使用比较频繁的字段建立索引
  • 尽量缩小子查询的范围
  • 避免改变索引列的类型

事务

  • 事务的基本要素

    • 原子性
    • 一致性
    • 隔离性
    • 持久性

  • 事务的隔离级别

    • 读未提交：事务最低的隔离级别，允许读取未提交的数据，可能会发生脏读、不可重复读、幻读。
    • 读提交：允许读取已经提交的数据，可能会发生不可重复读、幻读。
    • 可重复读：对同一字段的多次读取结果是一致的，可能会发生幻读。
    • 串行化：最高的隔离级别，事务按照一定顺序依次执行，可防止脏读、不可重复读、幻读。
    事务隔离级别 脏读 不可重复读 幻读
    读未提交 是 是 是
    读提交 否 是 是
    可重复读 否 否 是
    串行化 否 否 否

  • 事务的并发问题

    • 脏读：事务A读取了事务B更新的数据，然后事务B回滚，那么事务A读取的数据就是脏数据。
    • 不可重复读：事务A多次读取同一数据，事务B在事务A读取的过程中，对该数据作了更新并提交，导致事务A每次读取的数据不一致。（侧重于对数据的修改）
    • 幻读：比如系统管理员A将表中所有学生的成绩从具体分数改为了ABCDE等级，但这个时候系统管理员B又插入了一条具体分数的数据，然后管理员A发现还有一条记录没有被修改，就好像发生幻觉一样（侧重于插入和删除记录）

面向对象基础

Java创建对象的几种方式

  • new

  • 反射

  • 使用clone（）方法

无论何时，我们调用一个对象的clone（）方法，jvm就会创建一个新的对象，并且将前面对象的内容拷贝进去。用clone（）方法创建对象不会调用任何构造方法。

  • 使用反序列化

如何比较两个Integer类型的数据？

使用“==”比较的是两个数据的地址值；

使用equals（）比较的是数据的内容；

因此一般使用equals（）来比较Integer类型的数据。

接口和抽象类的异同？

共同点：

  • 都不能实例化
  • 都可以有默认的实现方法
  • 都可以有抽象类

不同点：

  • 一个类只能继承一个抽象类，可以实现多个接口。
  • 接口中的成员变量必须是public static final类型的，必须有初始值且不能修改。抽象类中的成员变量默认为default类型，可以在子类中重新定义和修改。

实际开发中，java使用什么数据类型来代表价格？

  • Java使用BigDecimal来表示，数据库使用decimal（5,2）。

  • BigDecimal是Java用于高精度浮点运算的关键类。

浅拷贝和深拷贝

  • 浅拷贝：对基本数据类型进行值传递，对引用数据类型进行引用传递。
  • 深拷贝：对基本数据进行值传递，对引用数据类型，创建一个新的对象，并将内容复制进去。

String

String、StringBuffer、StringBuilder的区别

  • String是不可变字符串，StringBuffer和StringBuilder是可变字符串。
  • StringBuffer是线程安全的（加了synchronized关键字），效率较低。
  • StringBuilder是线程不安全的，效率较高。

String s1 = new String(“abc”)；这句话创建了几个对象？

一个或两个。

异常

Exception和Error的区别？

  • Exception：程序本身可以处理的异常。可以用catch捕获，可以分为CheckedException（受检查异常，必须处理），UncheckedException（不受检查异常，不必处理）。
  • Error：程序无法处理的错误。

try-catch-finally方法有return如何执行？

当try或者catch中有return时，先执行finally，在执行return。

List

ArrayList和LinkedList的区别?
  • ArrayList是基于动态数组的数据结构，LinkedList是基于链表的数据结构。
  • ArrayList更适合修改操作，LinkedList适合增删操作，但实际开发中，基本都使用ArrayList。
ArrayList和Array(数组)的区别?
  • ArrayList基于动态数组实现，Array基于静态数组实现。
  • ArrayList可以扩容或缩容，Array不能改变存储长度。
  • ArraryList可以使用泛型，Array不可以。
  • ArrayList只能存储对象或者基本数据类型对应的包装类，Array可以存储对象，也可以存储基本数据类型。
  • ArrayList支持插入、删除、遍历等操作，并提供了API操作方法，如add（）、remove（）、set（）。Array是固定长度数组，只能根据下标来访问数组内元素。
  • ArrayList创建时不需要指定大小，Array必须指定大小。
ArrayList可以添加null值吗？

ArraryList可以添加任意类型的数据，但不建议添加null值，会导致空指针异常。

ArrayList插入和删除的时间复杂度？

对于插入：

  • 头部插入，时间复杂度为O(n)。
  • 尾部插入，如果ArrayList的容量未达到极限，时间复杂度为O(1)，如果达到极限，时间复杂度为O(n)+O(1)。
  • 指定位置插入，时间复杂度为O(n)。

对于删除：

  • 头部删除，时间复杂度为O(n)。
  • 尾部删除，时间复杂度为O(1)。
  • 指定位置删除，时间复杂度为O(n)。
LinkedList插入和删除的时间复杂度？
  • 头部插入/删除，只需要修改头节点的指针，时间复杂度为O(1)。
  • 尾部插入/删除，只需要修改尾节点的指针，时间复杂度为O(1)。
  • 指定位置插入/删除，先移动到指定节点，在修改该节点的指针，时间复杂度为O(n)
Comparable和Comparator接口的区别？

都是java中用于排序的接口。

  • Comparable中的排序方法是compareTo(Object object)。
  • Comparator中的排序方法是compare(Object obj1,Object obj2)。

Set

HashSet、LinkedHashSet和TreeSet三者的异同？
  • 都继承Set接口，保证元素唯一不重复，都不是线程安全的。
  • 主要区别在于三者数据结构不同，HashSet基于哈希表的数据结构（通过HashMap实现），LinkedHashSet基于链表+哈希表的数据结构，TreeSet基于红黑树的数据结构。
  • 数据结构的不同导致应用场景不同。HashSet用于不需要保证元素插入和取出顺序的场景，LinkedHashSet用于保证元素的插入和取出顺序保证FIFO（先进先出）的场景，TreeSet用于支持对元素自定义排序的场景。

Map

HashMap和HashTable的区别？

  • 线程安全性和效率：HashMap线程不安全 效率高，HashTable线程安全效率低，实际开发中，HashTable基本被淘汰。

  • 对null值的支持：HashMap可以存储null值，key只能有一个null，value可以有多个null。HashTable不支持null值。

  • 底层数据结构：JDK1.8之后，HashMap在解决哈希冲突时多了个机制，当链表长度大于阈值（默认8）时，将链表转换为红黑树，以减少搜索时间。HashTable没有这个机制。

HashMap和HashSet的区别？

HashMap和TreeMap的区别？

HashMap比TreeMap多了对集合中元素根据key键排序的能力和对集合内元素搜索的能力。

HashSet如何检查重复？

在将对象加入HashSet时，会先计算这个对象的hashcode判断对象加入的位置，然后跟其他对象的hashcode作比较，如果没有相符的hashcode，HashSet会假设么有重复出现。但是如果发现有相同的hashcode，会调用equals( )方法检查hashcode相等的对象是否真的相同。若二者相同，就不会加入该对象。

在jdk1.8之后，无论是否存在相同元素，HashSet都会直接插入，只是会在add( )方法的返回值处告诉我们插入之前是否存在相同元素。

HashMap 多线程操作导致死循环问题

并发环境下，推荐使用ConcurrentMap。

HashMap为什么线程不安全？

多线程环境下，多个线程对HashMap执行put( )操作，可能会造成数据覆盖。

举个例子：

  • 线程1和线程2同时执行put( )操作，并且发生了哈希冲突(hash函数计算出的插入下标是相同的)。
  • 不同的线程可能在不同的时间片获取cpu执行的机会，当线程1执行完哈希冲突判断后，由于时间片耗尽挂起，线程2先完成了插入操作。
  • 随后，线程1获取时间片，因为已经判断没有哈希冲突，所以线程1会直接进行插入操作，这样线程1插入的数据覆盖掉线程2的数据。
HashMap常见的遍历方式？
  • 1.迭代器(Iterator)方式遍历
  • For Each方式遍历
  • Lambda表达式遍历
  • Stream API遍历
ConcurrentHashMap和HashTable的区别？

主要体现在实现线程安全的方式上不同。

  • 底层数据结构不同

并发编程

volatile关键字

如何保证变量的可见性？

volatile关键字可以保证变量的可见性，如果将变量声明为volatile，这就指示JVM，这个变量是共享且不稳定的，每次使用它都到主存中获取。

如何禁止指令重排

在 Java 中，volatile 关键字除了可以保证变量的可见性，还有一个重要的作用就是防止 JVM 的指令重排序。 如果我们将变量声明为 volatile ，在对这个变量进行读写操作的时候，会通过插入特定的内存屏障的方式来禁止指令重排序。

synchronized关键字

synchronized 关键字的使用方式主要有下面 3 种：

  1. 修饰实例方法
  2. 修饰静态方法
  3. 修饰代码块
synchronized关键字和volatile的区别？
  • volatile的性能优于synchronized。
  • volatile只能修饰变量，synchronized还可以修饰方法和代码块。
  • volatile只能保证数据的可见性，不能保证数据的原子性。synchronized二者都可以保证。
  • volatile主要用于处理变量在多线程之间的可见性，synchronized主要用来解决多线程之间访问资源的同步性。

悲观锁和乐观锁

悲观锁：

总是假设最坏的情况，每次去拿数据的时候，认为数据会被别人修改，所以每次拿数据的时候都会上锁。像 Java 中synchronized和ReentrantLock等独占锁就是悲观锁思想的实现。

乐观锁：

总是假设最好的情况，每次拿数据，数据不会被别人修改，所以不会上锁；

但是在进行更新操作的时候，会判断在此期间别人有没有更新这个数据，通常使用版本号version来实现。

悲观锁和乐观锁的使用场景：

  • 乐观锁适用于少写的情况下
  • 悲观锁适用于多写的情况下

什么是线程死锁？如何预防和避免死锁？

什么是死锁：

一组进程互相等待对方持有资源，导致不能继续执行下去的情况。

死锁发生四个必要条件：

  • 互斥条件：一段时间内某资源只能被一个进程占用，其他进程等待。
  • 请求和保持条件：进程已经保持至少一个资源，此时又发出了新的请求，而该资源又被其他进程占用，手里的资源又不放掉。
  • 不剥夺条件：进程已获得的资源，在未使用完成前，不能被剥夺。
  • 环路等待条件：进程之间形成获取资源的环形链。

如何预防死锁？

  • 1.破坏请求和保持条件：一次性申请所有资源。
  • 2.破坏不剥夺条件：占用部分资源的线程在请求其他资源时，如果请求不到，就将已占有的资源释放。
  • 3.破坏循环等待条件：线程按照顺序来请求资源。

如何避免死锁？

在资源分配时，借助算法对资源分配进行计算评估，使系统进入安全状态。

可以直接调用Thread类的run( )方法吗？

不可以。调用start( )方法可以启动线程使线程进入就绪状态，可以实现多线程。直接调用run( )方法，会在main方法中将run( )方法当做一个普通方法在主线程下执行，不能实现多线程。

sleep( )和wait( )方法的区别？

  • sleep( )是Thread类下的静态方法，wait是Object类的方法。

  • sleep( )不会释放锁；wait( )会释放锁。

  • wait( )方法调用后，线程不会自动苏醒，需要调用notify( )或notifyAll( )方法来唤醒。sleep( )方法执行完，线程会自动苏醒。

synchronized和lock锁的区别

  1. synchronized是java内置的关键字，Lock是java接口
  2. synchronized无法判断获取锁的状态，Lock可以判断是否获取了锁
  3. synchronized会自动释放锁，Lock必须手动释放，若不释放，就会产生死锁。synchronized 线程1（获取锁）、线程2（等待）；Lock锁不一定会一直等待，可以调用lock.tryLock（）尝试去获取锁，获取不到就结束等待。
  4. synchronized适合锁少量的同步代码；lock适合锁大量的同步代码。

反射

  • 什么是反射

    java的反射机制是指在java运行状态下，能够获取任意类的属性和方法，能够调用任意对象的方法。这种动态获取类信息和调用方法的功能叫做反射。

  • 反射使用步骤

    • 1.获取要操作类的Class对象
    • 2.调用Class类中的方法
    • 3.使用反射API类操作

  • 获取Class对象的3种方法

    • 调用某个对象的getClass( )方法

    • 调用某个类的class属性

    • 使用Class类中的forName( )静态方法（最安全、性能最好）

  • 反射API类

    • Class类：反射核心类，可以获取类的属性、方法等信息
    • Filed类：表示类的成员变量，可以获取和设置类中的属性值
    • Method类：表示类的方法，可以获取类中的方法信息或者执行方法
    • Constructor：表示类的构造器

AOP

面向切面编程（AOP）的设计模式，在该模式中，程序被分为核心业务逻辑和横切关注点两个部分。

切面类可以封装横切关注点的逻辑，将其与核心业务逻辑分隔开。

使用场景有：

  • 日志记录
  • 事务管理
  • 缓存管理
  • 权限控制
  • 异常处理
