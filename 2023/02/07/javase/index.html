方法的重写和重载：

重写：

   1.重写的方法与父类方法有相同的方法名、参数列表。

   2.子类重写方法返回值类型不能大于父类方法。

   3.子类重写方法访问权限不能小于父类方法。

重载：

   1.方法名相同，参数列表不同，不看返回值类型。

static关键字：

静态变量特点：

             1.静态变量值为所有对象共享。

             2.如果权限修饰符允许，可通过"类名.变量名"访问。

             3.静态变量在本类中，可以在任意方法、代码块、构造器中直接使用。

多线程：

    1.线程的创建方式：
       1.1 继承Thread类：
         (1)定义Thread类的子类，重写run()方法，run()方法内容即为线程执行内容。
         (2)创建Thread子类的实例，即创建线程对象。
         (3)调用线程对象的start()方法，启动线程。
       1.2 实现Runnable接口：
         (1)定义Runnable接口的实现类，并重写该接口的run()方法.
         (2)创建Runnable实现类的实例，并以此实例作为Thread的target参数来创建Thread对象，该Thread对象才是真正的线程对象。
         (3)调用线程对象的start()方法，启动线程。调用Runnable接口实现类的run方法。
    2.线程的生命周期：
       新建、就绪、运行、阻塞、死亡
    3.线程的安全问题
       3.1 同一资源问题和线程安全问题
       	  3.1.1 局部变量不能共享
       	  案例：火车站卖票，三个窗口同时卖100张票，不能错票、重票。
       	  实例代码：
       	  	package com.atguigu.unsafe;

            class Window extends Thread {
                public void run() {
                    int ticket = 100;
                    while (ticket > 0) {
                    System.out.println(getName() + "卖出一张票，票号:" + ticket);
                    ticket--;
                    }
                }		
            }

            public class SaleTicketDemo1 {
                public static void main(String[] args) {
                    Window w1 = new Window();
                    Window w2 = new Window();
                    Window w3 = new Window();

                    w1.setName("窗口1");
                    w2.setName("窗口2");
                    w3.setName("窗口3");

                    w1.start();
                    w2.start();
                    w3.start();
                }
            }
           结果：发现卖出300张票。
           问题：局部变量是每次调用方法都是独立的，那么每个线程的run()的ticket是独立的，不是共享数据。

           3.1.2 不同对象的实例变量不能共享
           示例代码：
           		package com.atguigu.unsafe;

                    class TicketWindow extends Thread {
                        private int ticket = 100;

                        public void run() {
                            while (ticket > 0) {
                                System.out.println(getName() + "卖出一张票，票号:" + ticket);
                                ticket--;
                            }
                        }
                    }

                    public class SaleTicketDemo2 {
                        public static void main(String[] args) {
                            TicketWindow w1 = new TicketWindow();
                            TicketWindow w2 = new TicketWindow();
                            TicketWindow w3 = new TicketWindow();

                            w1.setName("窗口1");
                            w2.setName("窗口2");
                            w3.setName("窗口3");

                            w1.start();
                            w2.start();
                            w3.start();
                        }
                    }
                    结果：发现卖出300张票。
                    问题：不同的实例对象的实例变量是独立的。
             3.1.3 静态变量是共享的
             示例代码
             		package com.atguigu.unsafe;

                    class TicketSaleThread extends Thread {
                        private static int ticket = 100;

                        public void run() {
                            while (ticket > 0) {
                                try {
                                    Thread.sleep(10);//加入这个，使得问题暴露的更明显
                                } catch (InterruptedException e) {
                                    e.printStackTrace();
                                }
                                System.out.println(getName() + "卖出一张票，票号:" + ticket);
                                ticket--;
                            }
                        }
                    }

                    public class SaleTicketDemo3 {
                        public static void main(String[] args) {
                            TicketSaleThread t1 = new TicketSaleThread();
                            TicketSaleThread t2 = new TicketSaleThread();
                            TicketSaleThread t3 = new TicketSaleThread();

                            t1.setName("窗口1");
                            t2.setName("窗口2");
                            t3.setName("窗口3");

                            t1.start();
                            t2.start();
                            t3.start();
                        }
                    }
                    结果：成功卖出100张票。

    4.同步机制解决线程安全问题
        根据案例简述：窗口1线程进入操作的时候，窗口2和窗口3线程只能在外等着，窗口1操作结束，
        窗口1和窗口2和窗口3才有机会进入代码去执行。也就是说在某个线程修改共享资源的时候，其
        他线程不能修改该资源，等待修改完毕同步之后，才能去抢夺CPU资源，完成对应的操作，保证
        了数据的同步性，解决了线程不安全的现象。

        4.1 同步机制解决线程安全问题原理
            同步机制的原理，其实就相当于给某段代码加“锁”，任何线程想要执行这段代码，都要先获得“锁”，我们称它为同步锁。最多只能有一个线程拿到锁。

区分String、StringBuffer、StringBuilder

  • String：不可变字符串
  • StringBuffer：可变字符串，方法使用synchronized修饰，线程安全，效率低。
  • StringBuilder：可变字符串，线程不安全，效率高。
